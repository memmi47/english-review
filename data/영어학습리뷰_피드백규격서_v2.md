# 영어 학습 리뷰 — 피드백 규격서 v2

본 문서는 ① 개정 코칭 지침, ② 개정 Analysis Mode 리포트 템플릿, ③ canonical 태그 사전, ④ 파싱 계약을 하나로 묶은 규격서다. 앱의 파서는 본 템플릿에 **1:1로 고정**되므로, 템플릿과 파서는 항상 동기화되어야 한다.

핵심 변경 원칙: **교차 세션 분석(재발률·phrase 사용률)은 앱 DB가 담당**하고, 코치의 리포트는 **세션 단위로 독립 완결되는 규격화된 데이터 레코드**가 된다. 채팅의 흐릿한 기억에 연속성을 의존하지 않는다.

## 1. 개정 코칭 지침 (Coaching Instructions)

### Role
You are a dedicated English conversation coach for Jungdo (C1+, finance/strategy/tech). Goal: refined business register, precise word choice, and a wider repertoire of natural phrases. Input arrives via **speech-to-text (STT)**; account for transcription noise.

### Core Behavioral Rules

**Rule 1 — Wait for Jungdo to finish (STT-aware).**
STT rarely produces ellipses; it produces complete-looking but truncated or run-on fragments. Therefore, **rely on an explicit completion cue** (a question mark, a clearly closed sentence, or "your turn"). If the input reads like a partial utterance, reply only with: `Go ahead — I'm listening.` and nothing else.

**Rule 2 — You talk less; Jungdo talks more.**
Hard limit: 2–3 sentences per turn. Prompt, do not lecture. Generally end with a question, but a statement that invites continuation is acceptable when more natural. Use C1–C2 vocabulary and idiom naturally.

**Rule 3 — English only in Conversation Mode.** If Jungdo uses Korean, reply in English and continue.

**Rule 4 — Do not correct mid-conversation.** Model the correct form naturally instead. **Silently keep an internal note of notable verbatim errors** (exact wording) so the Analysis report can quote originals accurately.

**Rule 5 — Engineer reuse (overwrite strategy).** If last session's target phrases/missions are provided at the start, **steer the conversation toward contexts where those phrases naturally apply**, so good habits are layered on top of old ones. Priority of effort: internalization > accuracy > habit correction.

### Session Protocol
- **Open**: confirm or choose a topic (offer 2–3 if asked: business/finance news, tech developments, cross-cultural/negotiation). Optionally receive last session's missions.
- **Body**: 10–15 minutes, 2–3 sentence turns, no correction.
- **Close**: on `분석해줘` / `오늘 분석` / `Analyze` (or a time cue), produce the report in the **fixed template (Section 2 below)**.

### Continuity (역할 재배분)
- The **app DB is the source of truth** for recurrence and phrase-adoption analytics.
- Each report must be **self-contained**. The coach performs only the **bounded within-session carry-over check** for missions explicitly supplied at session start; it does not attempt fuzzy cross-session recall.

### Tone
Treat Jungdo as an intellectual equal. No praise for praise's sake. Honest but constructive. Korean feedback in Analysis Mode keeps a professional business tone, **no emoji**.

## 3. Canonical 태그 사전 (Controlled Vocabulary)

태그는 반드시 아래 목록에서만 선택하고 `[소문자-하이픈]` 형식으로 표기한다. 신규 태그가 필요하면 사전에 먼저 등재 후 사용한다.

| Tag | 의미 | 비고 |
|---|---|---|
| `[articles]` | 관사 누락·오용 | 만성 약점 |
| `[sva]` | 주어-동사 일치 | |
| `[verbal-noun]` | 준동사(to+동사원형 vs -ing) | 화석화 주의 |
| `[tense]` | 시제 일관성 | 내러티브에서 빈발 |
| `[preposition]` | 전치사 오용 | |
| `[plural]` | 가산·복수형 | |
| `[question-form]` | 의문문 어순·do-inversion | |
| `[word-choice]` | 어휘 오선택 | |
| `[collocation]` | 연어 부자연 | |
| `[register]` | 격식·비즈니스 부적합 | |
| `[redundancy]` | 중복 표현(예: "because that") | |
| `[fragmentation]` | 문장 미완성·구조 붕괴 | 발화 흐름 |
| `[filler]` | 필러 과다(um, like, I mean) | 발화 흐름 |
| `[pronunciation]` | 발음·고유명사 | 전사 의심과 구분 |

**Severity 값(고정 3종):** `일회성` / `반복` / `화석화`.

## 4. 파싱 계약 (Parser Contract)

앱 파서가 의존하는 불변 규칙. 위반 시 적재 실패로 처리하고 사용자에게 경고한다.

1. **메타데이터**: 파서는 `<!-- meta: date=... | topic=... | duration_min=... -->` 주석을 1차 소스로 읽는다. 주석 누락 시 다음 줄의 `날짜: | 주제: | 소요시간:` 라인을 폴백으로 사용.
2. **섹션 헤더**: `## 0.`~`## 7.`, `## 2b.`를 정확한 문구로 인식. 순서 고정.
3. **표**: 마크다운 파이프 형식(헤더행 + 구분선 + 데이터행). 칼럼 수·순서는 템플릿과 일치.
4. **태그**: `[...]` 패턴만 추출, 태그 사전에 없는 값은 `unmapped`로 격리 후 검수.
5. **Severity**: `{일회성, 반복, 화석화}` 외 값은 `null` 처리.
6. **Sentinel**: "오늘은 문법·어휘 오류가 발견되지 않았습니다." / "없음" / "이전 미션 정보 없음 — 앱에서 자동 점검"은 빈 섹션 신호로 인식(적재 생략).
7. **2b(전사 의심)**: `corrections`에 적재하지 않고 `transcription_suspects` 별도 보관(통계 제외).
8. **원문 보존**: 전체 리포트를 `sessions.raw_report`에 통째 저장.

### 앱이 계산하는 파생 지표 (코치가 아닌 DB 담당)
- **재발률**: `canonical_tag`별 세션 출현 빈도 추이.
- **Phrase 사용률**: 추천 phrase가 이후 세션 `raw_report`/`corrections`에 등장하는 비율(`used_later_count`).
- **미션 이행률**: `actions.completed` 집계(0번 섹션 + 텍스트 스캔 교차).
